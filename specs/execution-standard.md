# 🔧 项目执行规范

## 1. 检索规范

### 1.1 检索频率
- 每 30 分钟执行一次 GitHub 检索
- 每次检索使用 10+ 组关键词，确保覆盖面

### 1.2 项目筛选标准
入选条件（满足任一）：
- ⭐ Star ≥ 50 且与 Claude Code 知识库相关
- ⭐ Star ≥ 10 且最近 30 天有更新
- 任何 Star 数但被多个相关项目引用

排除条件：
- 纯文档项目（无代码）
- 已归档（Archived）
- 超过 1 年无更新且 Star < 100

### 1.3 去重规则
- 以 `owner/repo` 为唯一标识
- 同一组织的相似项目合并分析

## 2. 调研规范

### 2.1 每个项目必须覆盖
- [ ] **项目概述**: 定位、目标用户、核心价值
- [ ] **架构分析**: 系统架构图、核心模块、数据流
- [ ] **技术栈**: 语言、框架、依赖、存储方案
- [ ] **源码分析**: 核心代码路径、关键算法、设计模式
- [ ] **优缺点**: 至少各列出 3 条
- [ ] **最佳场景**: 至少列出 3 个适用场景
- [ ] **与 Claude Code 的集成方式**: MCP / API / CLI / 直接集成
- [ ] **性能数据**: 索引速度、检索延迟、准确率（如有）

### 2.2 文档格式
- 使用 Markdown 格式
- 代码块标注语言类型
- 架构图使用 Mermaid 或 ASCII
- 表格对比关键指标

## 3. 落地验证规范

### 3.1 Godot 验证流程
1. 克隆 Godot 最新稳定版源码
2. 按项目文档安装部署知识库方案
3. 索引 Godot 源码
4. 执行 Benchmark 测试
5. 记录全程问题和解决方案

### 3.2 必须记录的坑点
- 安装过程中的兼容性问题
- 配置参数的最佳实践
- 索引大代码库的性能瓶颈
- 检索质量调优经验
- 与 Claude Code 集成的注意事项

## 4. 文档管理规范

### 4.1 README 维护
- 根目录 README 仅作为**索引页**
- 实时更新项目状态（⏳🔍✅🚫）
- 更新日志记录每次变更

### 4.2 目录命名
- 项目目录: `projects/<org>-<repo>/`（如 `projects/anthropic-mcp/`）
- 使用小写 + 短横线

## 5. Git 提交规范

### 5.1 Commit Message 格式
```
<type>: <subject>

[optional body]
```

Type:
- `🔍 discover`: 发现新项目
- `📝 analysis`: 源码分析更新
- `✅ complete`: 调研完成
- `🧪 benchmark`: Benchmark 测试结果
- `🐛 pitfall`: 踩坑记录
- `📐 spec`: 规范更新
- `🔧 script`: 脚本/工具更新

## 6. 问题总结规范

遇到的问题统一记录到 `specs/lessons-learned.md`，格式：
```markdown
### [日期] 问题标题
- **现象**: ...
- **原因**: ...
- **解决方案**: ...
- **预防措施**: ...
- **影响范围**: ...
```
